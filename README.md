# ClarityPay Take-Home: Consumer Credit Analytics

**Question**: *"Are we lending to the right borrowers?"*
**Dataset**: Lending Club 2007–2020 Q3 (Kaggle)

---

## Project Structure

```
ClarityPay/
├── analysis.ipynb          # Main Jupyter Notebook (all analysis)
├── slides_v2.pptx          # 8-slide presentation deck
├── build_slides.py         # Script that generates slides_v2.pptx from figures/
├── figures/                # All charts generated by the notebook
├── archive/
│   ├── Loan_status_2007-2020Q3.gzip   # Raw data (~1.7 GB, NOT in repo — see below)
│   └── LCDataDictionary.xlsx          # Field definitions (151 columns)
└── README.md               # This file
```

> **Data file not included**: `archive/Loan_status_2007-2020Q3.gzip` (~1.7 GB) exceeds GitHub's 100 MB file limit and is excluded via `.gitignore`. Download it from [Kaggle — Lending Club Loan Data](https://www.kaggle.com/datasets/ethon0426/lending-club-20072020q1) and place it at `archive/Loan_status_2007-2020Q3.gzip` before running the notebook.

## How to Run

**Requirements**: Python 3.10+, and the following packages:

```
pandas >= 1.5
numpy
matplotlib
seaborn
openpyxl        # for reading .xlsx data dictionary
python-pptx     # for generating slides.pptx
jupyter
```

Install with:

```bash
pip install pandas numpy matplotlib seaborn openpyxl python-pptx jupyter
```

**Run the notebook**:

```bash
jupyter notebook analysis.ipynb
```

Then run all cells top-to-bottom (`Cell → Run All`).

> The data loading step (Phase 1) processes the 1.7 GB file in chunks and takes approximately 3–5 minutes depending on hardware. All subsequent cells run in seconds.

**Regenerate the slide deck** (after running the notebook):

```bash
python3 build_slides.py
```

---

## What the Notebook Does

| Phase | Content |
|---|---|
| 1 · Data Loading & Cleaning | Chunked read of gzip CSV; filter to completed loans; clean string columns; create target variable |
| 2 · EDA | Outcome distribution; missing values; volume/bad rate by year; univariate risk charts for 8 attributes |
| 3 · Interaction Analysis | FICO × DTI and FICO × Revol Util heatmaps to identify compounding risk |
| 4 · Policy Rules & Swap Sets | Define 8 individual rules + 6 rulesets; evaluate each with full trade-off metrics |
| 5 · Trade-off Visualization | Efficient frontier scatter plot; ruleset comparison chart |
| 7 · Summary & Recommendation | Final policy recommendation with exact metrics |

## Key Findings (Summary)

- **Overall bad rate**: 19.5% of completed loans charged off
- **Top 3 risk drivers**: FICO score, Lending Club loan grade, revolving credit utilization
- **Recommended policy** (FICO < 680 OR Grade E/F/G OR Revol Util > 80%):
  - Bad rate reduction: 4.0 pp (20.5% relative)
  - New bad rate: 15.5%
  - Volume declined: 43.3%
  - Good loans rejected: 40.5%
